# Domain → Tech Interfaces

> A bridge from domain knowledge to technical implementation

## Overview

These interface files help AI and developers understand:
- How domain requirements map to technical implementations
- Recommended software-skills combinations
- Common working patterns
- Anti-patterns to avoid

## Usage

### For AI Agents (recommend_skills)

After the MatchingEngine matches a domain skill, refer to the corresponding interface file to:
1. Determine which software-skills are needed
2. Choose the appropriate pattern
3. Recommend a technology stack

### For Developers

Understand which technical capabilities a given domain task requires, and plan a learning path.

## File Structure

Each interface file contains:

```markdown
# Domain → Tech Interface

## Domain Skills Covered
[List of covered domain skills]

## Requirement → Technology Mapping
[Table mapping requirements to technology]

## Common Combination Patterns
[Common skill combination patterns]

## Technology Stack Recommendations
[Technology stack recommendations]

## Anti-Patterns to Avoid
[Anti-patterns to avoid]
```
