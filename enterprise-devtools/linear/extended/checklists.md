# Linear — Extended Checklists

## Workflow Hygiene Checklist

- [ ] Workflow states kept close to Linear's defaults; custom states added sparingly and periodically pruned if unused
- [ ] Triage queue has a clear owner and a target processing SLA
- [ ] Triage queue length monitored as a routine hygiene metric, not just cleaned up occasionally
- [ ] Cycle scope set deliberately near the start; additions during an active cycle tracked and discussed explicitly
- [ ] Velocity/cycle completion trends reviewed over multiple cycles, not reacted to on a single noisy data point
- [ ] Sub-issue hierarchies actively maintained as real scope changes, not created once and left stale
- [ ] Branch-naming/PR-linking conventions documented and consistently followed across the team

## Integration & Reporting Checklist

- [ ] GitHub/GitLab auto-transition behavior spot-checked periodically against actual completed work
- [ ] Roadmap/projects used for cross-cycle initiatives, keeping cycles focused on current time-boxed scope
- [ ] Labels used consistently for cross-cutting concerns (bug vs. feature, team, priority) to keep filtered views useful
- [ ] Linear API/webhook integrations documented with an owner, not left as an undocumented dependency
- [ ] Import from a prior tool (Jira, Asana, Trello) reviewed for data fidelity before fully cutting over
