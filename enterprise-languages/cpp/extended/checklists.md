# C++ — Extended Checklists

## Safety Checklist

- [ ] No function returns a reference/pointer to a local (stack) variable
- [ ] Ownership expressed via `unique_ptr`/`shared_ptr`, not raw `new`/`delete`
- [ ] AddressSanitizer and UndefinedBehaviorSanitizer run in CI, not just ad hoc local debugging
- [ ] No reliance on signed integer overflow wrapping; unsigned types or overflow-checked arithmetic used where wraparound is intentional
- [ ] Polymorphic types passed/stored by reference or pointer, never by value (avoids slicing)
- [ ] Iterator invalidation rules checked for any container modified during iteration
- [ ] ThreadSanitizer run for any code sharing state across threads
- [ ] `-Wall -Wextra` (or equivalent) enabled and treated as build-blocking, not advisory

## Build Consistency Checklist

- [ ] Compiler flags consistent across all translation units in a build (single source of truth via CMake)
- [ ] Header/library versions consistent across the entire linked binary
- [ ] clang-tidy or Cppcheck run in CI for static analysis coverage beyond compiler warnings
- [ ] Rule of Zero followed where possible — special member functions generated by the compiler via RAII-managing members
- [ ] Move constructors/assignment marked `noexcept` where legitimately non-throwing, since this affects standard library optimization decisions
- [ ] Build reproducibility verified (clean build from scratch matches incremental build behavior)
