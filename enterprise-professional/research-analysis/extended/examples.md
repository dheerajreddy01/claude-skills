# Research Analysis Examples

## Hypothesis-Driven Analysis

### Hypothesis Formula

```
IF [we take action X]
THEN [it will produce result Y]
BECAUSE [based on reason/data Z]
```

### Hypothesis Example

```
IF we shorten the checkout flow to 3 steps
THEN cart conversion rate will increase by 20%
BECAUSE 60% of cart abandonment currently happens at steps 4-5
```

### Hypothesis Validation Process

1. **Form a hypothesis** - Propose a testable hypothesis based on preliminary observation
2. **Design the validation method** - Decide what data is needed and how to collect it
3. **Collect evidence** - Gather data that supports or refutes the hypothesis
4. **Analyze results** - Does the evidence support the hypothesis?
5. **Refine or confirm**
   - Hypothesis confirmed → move to the recommendation stage
   - Hypothesis not confirmed → refine the hypothesis and retest

## Triangulation Example

### Definition

Use multiple data sources or methods to validate the same conclusion, increasing research credibility

### Types of Triangulation

| Type | Description | Example |
|------|------|------|
| **Data triangulation** | Multiple data sources | Interviews + surveys + behavioral data |
| **Method triangulation** | Multiple research methods | Qualitative + quantitative |
| **Researcher triangulation** | Multiple independent analysts | Two analysts coding independently |
| **Theory triangulation** | Multiple theoretical frameworks | Behavioral economics + psychology |

### Applied Example

Research finding: "Users don't use the search feature"

- Validation 1 (behavioral data): only 5% of users use search
- Validation 2 (interviews): users said "I didn't know there was a search feature"
- Validation 3 (eye tracking): no one looks at the search box

→ Triangulation supports: the search feature has poor visibility

## Detailed Questionnaire Design Guide

### Questionnaire Structure Principles

**Structure layout:**
1. Start: simple, interesting questions (build trust)
2. Middle: core questions (the most important survey content)
3. End: demographic questions (less sensitive)

**Question design principles:**
- One question per topic
- Avoid leading questions
- Provide a "not applicable/no opinion" option
- Keep options mutually exclusive and exhaustive

**Length control:**
- 5-10 minutes is ideal (about 15-25 questions)
- Over 15 minutes, completion rates drop significantly

### Choosing Question Types

| Type | Applicable scenario | Example |
|------|----------|------|
| **Single choice** | Mutually exclusive options | Gender, age range |
| **Multiple choice** | Multiple selections allowed | Features used |
| **Scale** | Attitude/satisfaction | 1-5 satisfaction rating |
| **Matrix** | Multiple items on the same scale | Rating multiple features |
| **Open-ended** | In-depth understanding | Improvement suggestions |
| **Ranking** | Priority | Ranking feature importance |

### Common Questionnaire Mistakes

| Mistake | Example | Fix |
|------|------|------|
| **Double-barreled question** | Are you satisfied with the price and quality? | Split into two questions |
| **Leading** | Do you agree this is the best choice? | What's your view on this choice? |
| **Assumed premise** | How many times do you use it per day? | First ask whether they use it at all |
| **Incomplete options** | Age: 18-25, 26-35 | Add <18, 35+ |
| **Jargon** | What's your NPS score? | Explain it in plain language |

## Data Visualization Guide

### Choosing a Chart Based on Purpose

**Comparison**
- Category comparison → bar chart
- Time comparison → line chart
- Proportion comparison → stacked bar chart

**Distribution**
- Univariate distribution → histogram, box plot
- Bivariate relationship → scatter plot
- Multivariate relationship → heat map

**Composition**
- Proportional composition → pie chart (< 6 categories), treemap
- Composition change over time → area chart

**Trend**
- Trend over time → line chart
- Multi-metric trend → multi-axis line chart

### Visualization Principles

**Do:**
- Title conveys the message the chart is meant to communicate
- Y-axis starts at 0 (unless there's a valid reason not to)
- Use consistent color coding
- Cite the data source and time range
- Remove unnecessary decoration (3D, gradients)

**Don't:**
- Truncate the Y-axis to exaggerate differences
- Use too many colors (more than 6)
- Use a pie chart with more than 6 categories
- Use 3D effects that distort proportions
- Omit legends and labels
